### 1.4 計算ロジックの概要

本計算は、標準的（一般的）なPER/PBR/Forward PERの定義に基づいており、以下の特徴があります：

1. **価格データ**: `close`（未調整終値）を使用
2. **EPS/BPS/予想EPS**: `profit/equity/forecast_profit`と補正後株数から再計算
3. **分割・併合の考慮**: FY期末→評価日までの分割倍率を計算して株数を補正
4. **株数の定義**: 発行済み株式数 - 自己株式数（ネット株数）

## 2. 計算ロジックの詳細

### 2.1 計算フロー

```
1. 価格データの取得（close: 未調整終値）
   ↓
2. FY実績データの取得（最新のFYデータ）
   ↓
3. FY期末のネット株数計算
   net_shares_fy = shares_outstanding - treasury_shares
   ↓
4. 分割倍率計算（FY期末→評価日）
   split_mult = ∏(1 / adjustment_factor)
   ↓
5. 評価日時点のネット株数（補正後）
   net_shares_at_price = net_shares_fy × split_mult
   ↓
6. 標準EPS/BPS/予想EPSの計算
   eps_std = profit / net_shares_at_price
   bps_std = equity / net_shares_at_price
   forecast_eps_std = forecast_profit / net_shares_at_price
   ↓
7. PER/PBR/Forward PERの計算
   PER = price / eps_std
   PBR = price / bps_std
   Forward PER = price / forecast_eps_std
```

### 2.2 重要な設計方針

1. **価格データ**: `close`（未調整終値）を使用することで、評価日時点の市場価格に一致
2. **EPS/BPS/予想EPSの再計算**: 財務データに記録されている値をそのまま使用せず、`profit/equity/forecast_profit`と補正後株数から再計算
3. **分割・併合の考慮**: FY期末から評価日までの分割倍率を計算し、株数を補正
4. **株数の定義**: 発行済み株式数 - 自己株式数（ネット株数）を使用
